JavaScript Basic

Fecha de publicación: 2/07/21

Introducción

Es realmente importante, entender el contexto de lo que vamos a trabajar y para eso es importante entrar en contexto sobre JavaScript: ¿Que es?, ¿Para que sirve?, y ¿Por qué existe?. Esto es para facilitar el entendimiento de el tema a trabajar, ya que facilita la estructuración o encapsulamiento de la información en nuestra memoria.

¿Que es JavaScript?

JavaScript es un lenguaje programación que permite la creación de un objeto sin definir primero su clase, es decir, se basa en prototipos. Este es un leguaje multiparadigma, de un solo hilo, dinámico, con soporte para programación orientada a objetos, imperativa y declarativa.

¿Para que sirve JavaScript?

Al ser este un lenguaje multiparadigma, se puede clasificar en programación funcional, programación basada en prototipos, imperativo e interpretativo.

Esto para entender que este lenguaje nos permite trabajarlo como un herramienta muy versatil en diferentes areas de desarrollo como por ejemplo: Desarrollo Web Frontend o Desarrollo Web Backend y algunos usos externos a la web como aplicaciónes de escritorio (mayoritariamente widgets).

¿Por qué existe JavaScript?

Este lenguaje de programación nace de la necesidad de hacer interactiva una página web. Ya que anteriormente las páginas eran solo informativas ya que no permitian ningun tipo de interacion activa por parte del usuario.

Elementos de un lenguaje de programación

En un lenguaje de probramación se necesitan dos componentes principales: datos y tareas.

Los datos es la información que se guardara en memoria y esta puede tener varios tipos como: números, strings (texto), boleanos, null o undefined(valores vacios), arrays, objetos, variables.

Las tareas son funciones que le asignamos al codigo para que haga algo con dichos datos.

¿Cómo se hace esto en JavaScript?

Tipo números: 28,35, 13...

Tipo string: "Cualquier tipo de texto", "4".

Tipo boleanos: true o false.

Valores vacios: null tipo null o undefined tipo undefined.

Arrays: ["texto", 4, "5", 64] tipo objeto.

Tipo objetos: {nombre : "tuNombre"}

Variables: var nombreDeLaVariable = valoreDeLaVariable;.

Variables

Declarar una variable: se asigna un espacio en memoria con un nombre determinado.

  var edad;

Inicializar una variable: se asigna un valor en memoria a ese nombre determinado.

Números

  var nombreDeVariable = 15;

String

  var nombreDeVariable = "Cualquier Texto";

Arrays

  var elementos = ["Computadora, Celular"]

Objeto

  var persona = {
    nombre : "nombreCualquiera",
    edad : 30,
  }
  persona

Funciones

Funciones declarativas

  function nombreDeLaFuncion(c,d) {
    return c + d;
  }
  nombreDeLaFuncion();

Funciones de expresión

  var = function nombreDeLaFuncion(c,d) {
    return c + d;
  }
  nombreDeLaFuncion();

Hoisting

El concepto de Hoisting fue pensado como una manera general de referirse a cómo funcionan los contextos de ejecución en JavaScript (específicamente las fases de creación y ejecución).

Una estricta definición de hoisting sugiere que las declaraciones de variables y funciones son físicamente movidas al comienzo del código, pero esto no es lo que ocurre en realidad.

Lo que sucede es que las declaraciones de variables y funciones son asignadas en memoria durante la fase de compilación, pero quedan exactamente en dónde las has escrito en el código.

Coherción

Coherción implicita

La Coherción implicita modifica la naturaleza del valor mediante el uso de un operador.

Ejemplo 1

  var = a 4 + "7" ;
  a = "47"

La naturaleza o tipo de a será string.

Ejemplo 2

  var b = 3 * "4";
  b = 12

La naturaleza o tipo de b será número.

Coherción explicita

La Coherción explicita modifica la naturaleza del valor mediante el uso de una funcion asignada.

Ejemplo 1

  d = 34;
  var e = String(d);
  e = "34"

La naturaleza o tipo de d es número, mientras que la de e será string.

Ejemplo 2

  f = "45";
  var g = Number(f);
  e = 45

La naturaleza o tipo de f es string, mientras que la de g será número.

Truthy & Falsy

El valor pasado como primer parámetro se convierte en un valor booleano, si es necesario. Si el valor se omite o es 0, -0, null, false, NaN, undefined, o la cadena vacía (" "), el objeto tiene un valor inicial de false.

Todos los demás valores, incluido cualquier objeto, un arreglo vacío ([ ]) o la cadena "false", crean un objeto con un valor inicial de true.

Valores false

  Boolean (0);   // return false
  Boolean (null);   // return false
  Boolean (NaN);   // return false
  Boolean (undefined);   // return false
  Boolean (false);   // return false
  Boolean ("")   // return false;

Valores true

  Boolean (" ");   // return true
  Boolean ("a");   // return true
  Boolean (1);   // return true
  Boolean ([ ]);   // return true
  Boolean ({});   // return true
  Boolean (true)   // return true;
